# Button Reply Rule

- Keep reply bodies short and plain.
- Never include labels like “Message”, emoji headers, sender labels, or JSON/button payloads in visible text.
- Use native interactive buttons only.
- Prefer one button per row when a label may need extra space.
- Keep button text short. Do not try to force wrap beyond what the client supports.
- If the platform controls wrapping, do not mention it in the visible message.
