# Research Context Cheat Sheet

A quick guide for choosing the right skill and output path.

## 1. Main entry point

### `research-context`
Use this first when the source can be:
- a newspaper screenshot
- an article clipping
- copied article text
- a PDF or scanned article
- a mixed-source article context

Use it when the required output is not yet fixed and you want a menu of options.

### Task menu inside `research-context`
1. Article Analysis
2. Investment Analysis
3. Combined Research Pack
4. LinkedIn Draft
5. WhatsApp Summary
6. Blog-Ready Article
7. Email Pack
8. Full Publishing Pack

## 2. Supporting subworkflows

### `article-analysis`
Use this when you want only source-grounded editorial or communication outputs.

Best for:
- key takeaways
- detailed breakdown
- blog-ready writing
- LinkedIn post draft
- WhatsApp summary
- evidence check

Do not use this when the goal is stock ideas, sector beneficiaries, or fund selection.

### `investment-analysis`
Use this when you want market implications from the source context.

Best for:
- investment thesis
- sector impact mapping
- beneficiary analysis
- watchlists by market-cap bucket
- risk view
- mutual fund ideas aligned with Primeidea rules

Do not use this when the goal is article-only summarization or blog writing without market research.

## 3. Fast selection guide

If Partha says:
- "Summarize this article" → use `article-analysis`
- "Give me investment ideas from this" → use `investment-analysis`
- "Do both" → use `research-context` → Combined Research Pack
- "Make this LinkedIn ready" → use `research-context` or `article-analysis`
- "Make this WhatsApp ready" → use `research-context` or `article-analysis`
- "Turn this into a blog post" → use `research-context` or `article-analysis`
- "Prepare everything from this source" → use `research-context` → Full Publishing Pack
- "Draft emails from this article" → use `research-context` → Email Pack

## 4. Standing rules

- Keep analysis grounded in the source unless investment validation requires external market checks.
- Do not add unsupported facts.
- If the article is incomplete or OCR is weak, ask for a clearer source.
- For blog workflows, choose exactly one approved category and add relevant tags.
- For investment outputs, use Regular plan framing only.
- Do not send external emails without confirmation for the current run unless approval is already explicit.

## 5. Recommended default

When in doubt, start with:
### `research-context`
It is the umbrella workflow and the safest default entry point.
