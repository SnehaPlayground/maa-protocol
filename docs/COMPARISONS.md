# Maa Protocol Comparisons

## Maa vs LangGraph / LangChain

Choose Maa when you want:
- stronger operator-facing runtime governance
- built-in approval and validation patterns
- tenant-aware controls and operational docs

Choose LangGraph/LangChain when you want:
- a broader ecosystem
- lower-level agent graph construction
- more community examples out of the box

## Maa vs CrewAI / AutoGen

Choose Maa when you want:
- explicit runtime governance
- approval gates
- canary, cost control, and failure-handling patterns in one place

Choose CrewAI/AutoGen when you want:
- faster experimentation
- lighter multi-agent prototyping

## Maa vs Temporal / Cadence

Choose Maa when you want:
- agent-specific orchestration patterns
- operator-facing guardrails and validation workflows

Choose Temporal/Cadence when you want:
- durable distributed workflows at larger infrastructure scale
- general workflow orchestration beyond agent systems

## Summary

Maa is best understood as a governance-heavy agent orchestration framework, not as a universal replacement for every orchestration stack.
